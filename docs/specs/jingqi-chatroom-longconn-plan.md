# 京奇聊天室长连接压测方案（2026-04-30）

## 1. 背景与目标
- 直播间数量：20 个
- 单房在线人数范围：2w–10w
- 重点关注：开播前 3–5 分钟瞬时涌入高峰

目标：
1. 验证瞬时涌入窗口内的连接稳定性（connect/login/join）。
2. 验证峰值阶段消息链路时延与错误率。
3. 识别容量边界与风险点（连接雪崩、重连风暴、延迟抖动）。

## 2. 压测模型
### 2.1 房间模型
- 房间总数固定：20
- 分层分布建议：
  - 热门房：8w–10w（约 20%）
  - 中腰部：5w（约 50%）
  - 长尾：2w（约 30%）

### 2.2 用户行为模型
- 只看不发言：90%
- 轻互动（低频消息）：9%
- 高互动（高频消息）：1%
- 基本流程：connect -> login -> join_room -> heartbeat -> recv/send

### 2.3 瞬时涌入模型
- 窗口：开播前 3–5 分钟
- 分段建议：
  - T-5~T-3：缓升
  - T-3~T-1：快速爬升
  - T-1~T：峰值冲顶

## 3. 执行阶段
1. 基线：低压稳态（2w 档）15–30 分钟。
2. 分档爬升：2w -> 5w -> 8w（每档保压 20 分钟）。
3. 瞬时涌入：3–5 分钟冲到峰值（含 10w 房间），重复 3 轮。
4. 峰值保压：30–60 分钟。
5. 回落：模拟开播后增速下降与部分离场，观察恢复能力。

## 4. 指标与验收
### 4.1 关键指标
- 连接链路：connect/login/join 成功率、断连率、重连率
- 消息链路：send 成功率、send_to_ack P50/P95/P99、end_to_end P50/P95/P99
- 稳定性：错误率、资源水位（CPU/内存/带宽/连接数）

### 4.2 建议阈值
- connect/login/join 成功率 >= 99.9%
- 峰值窗口错误率 < 0.5%
- P99 时延满足业务 SLO
- 无持续性重连风暴与消息堆积

## 5. 配置字段落地（config.yaml）
字段已落地在：
- `locust.scenes[].name == "jingqi-chatroom"`

字段说明：
1. `room_count`
- 直播间数量（本次为 20）
2. `room_tiers`
- 房间分层定义（按房间占比）与各层目标人数
3. `surge_window_s`
- 瞬时涌入窗口时长（秒）
4. `surge_curve`
- 窗口内涌入曲线（`linear` / `s_curve` / `front_loaded`）
5. `peak_target_users`
- 峰值在线目标（可为总量或分房目标）

## 6. 备注
- 长连接连接方式以 `locust.<active_env>.mode + use_ssl` 为准。
- 当前方案仅定义压测建模与字段，不包含业务脚本实现细节。
